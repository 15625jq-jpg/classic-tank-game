# 经典坦克大战平衡版发布与更新说明

## 1. 当前发布结构

为兼顾“正式入口”和“后续持续更新”，当前项目已补充以下发布文件：

- `index.html`：正式首页
- `play/index.html`：正式游玩入口
- `updates/index.html`：近期版本更新页
- `经典坦克大战平衡版版本调整说明.md`：完整历史版本记录

当前正式游玩入口默认跳转到：

- `经典坦克大战平衡版v3.8.html`

## 2. 推荐发布方式

当前项目最适合直接部署为静态站点，优先推荐：

1. `GitHub Pages`
2. `Cloudflare Pages`
3. `itch.io`（适合作为正式试玩页）

如果只选一种长期维护方式，优先建议 `GitHub Pages`。

## 3. 后续更新时怎么改

假设你后面新增了：

- `经典坦克大战平衡版v3.9.html`

建议按下面顺序更新：

1. 保留旧版本文件，不删除历史版本
2. 修改 `play/index.html`
   - 将跳转目标从 `../经典坦克大战平衡版v3.8.html`
   - 改为 `../经典坦克大战平衡版v3.9.html`
3. 修改 `index.html`
   - 更新首页显示的当前版本号
   - 更新首页“最近版本变化”区域
4. 修改 `updates/index.html`
   - 在顶部追加 `v3.9` 的更新摘要
5. 修改 `经典坦克大战平衡版版本调整说明.md`
   - 继续在末尾追加详细版本记录

这样做的好处是：

- 正式入口地址可以保持稳定
- 历史版本文件完整保留
- 首页、游玩页和更新页职责清晰
- 后续迁移到其他静态平台也比较方便

## 4. 发布前建议检查

每次发布前，建议至少确认：

- 最新版本 HTML 可以正常打开
- `play/index.html` 已指向最新版本
- 首页版本号与更新摘要已同步
- 更新页与完整版本说明已同步

## 5. 说明

当前这套结构采取的是“最小必要增量”方案：

- 不改动你现有的版本命名习惯
- 不重排原有历史文件
- 只额外补正式发布入口和更新展示层

这样更适合你继续按 `v3.9`、`v4.0` 的方式往后迭代。
